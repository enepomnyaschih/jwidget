jwidget/ReadonlyBindableMap

Consumption

import ReadonlyBindableMap from "jwidget/ReadonlyBindableMap";

Default export#

Hierarchy#

  • interface ReadonlyBindableMap<K, V>
  • interface DestroyableReadonlyBindableMap<K, V>
  • interface IBindableMap<K, V>
  • class BindableMap<K, V>

Description#

K
Key type.
V
Value type.

Bindable readonly wrapper over a native map.

Fields#

silent#

readonly silent: boolean

Checks if this collection never dispatches messages. This knowledge may help you do certain code optimizations.
size#

readonly size: Bindable<number>

Property containing number of entries in the map.
native#

readonly native: ReadonlyMap<K, V>

Native map - internal collection representation.
onSplice#

readonly onSplice: Listenable<IBindableMap.SpliceResult<K, V>>

Entries are deleted from the map and/or entries are added/updated in the map.
onReindex#

readonly onReindex: Listenable<ReadonlyMap<K, K>>

Keys of entries are changed in the map. Passes mapping of changed keys (old to new) as a message.
onClear#

readonly onClear: Listenable<ReadonlyMap<K, V>>

The map is cleared. Passes old map contents as a message.
onChange#

readonly onChange: Listenable<void>

The map is changed. Dispatched right after any another message.

detectSplice#

(newContents: ReadonlyMap<K, V>): IBindableMap.SpliceParams<K, V>

newContents
New map contents.
returns
splice method arguments. If no method call required, returns undefined.

Detects splice method arguments to adjust the map contents to newContents. Determines keys to be deleted and entries added/updated.

detectReindex#

(newContents: ReadonlyMap<K, V>): ReadonlyMap<K, K>

newContents
New map contents.
returns
keyMapping argument of reindex method. If no method call required, returns undefined.

Detects reindex method arguments to adjust the map contents to newContents. Determines new keys to be assigned to all entries. All values in newContents must be unique, and they must match values of this map.
